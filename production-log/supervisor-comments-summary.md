# Supervisor Comments Summary

## Summary of Comments

The supervisor's feedback suggested that the original comparison risked becoming too linear if it only compared model accuracy using MAE or RMSE. The project should show more critical thinking by comparing models across several dimensions, not only numerical error.

## Main Issues Identified

1. The phrase "comparative analysis" needs clearer meaning.
2. Model comparison should include accuracy metrics such as MAE, MSE, and RMSE.
3. The project should also consider interpretability, because some machine learning models may be difficult to explain in a small-scale EPQ.
4. The literature review may be the best place to discuss interpretability if it cannot be fully tested in the model implementation.
5. The aims and objectives may need to be revised so they match the expanded comparison.

## How I Will Respond

I will revise the project so that the main comparison has four dimensions:

| Dimension | Evidence to use |
| --- | --- |
| Accuracy | MAE, MSE, RMSE on the test set |
| Interpretability | Discussion of how easily each model's logic can be explained |
| Computational practicality | Notes on setup difficulty, training time, and reproducibility |
| Risk-management usefulness | Whether the forecast could support a practical volatility/risk decision |

This means the project will not simply claim that the model with the lowest error is automatically best. Instead, it will evaluate whether a model's accuracy improvement is worth the added complexity.

## Change to Aims and Objectives

The aim has been revised from "compare forecasting accuracy" to "critically evaluate predictive power, interpretability, computational practicality, and practical usefulness." This better matches the supervisor's feedback and gives the final report a stronger evaluative structure.

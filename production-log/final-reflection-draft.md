# Final Reflection Draft

## What I Learned About Research

The project showed that a strong investigation is not only about collecting sources but also about narrowing the question so that the comparison stays fair and manageable. It also showed the importance of keeping a critical perspective rather than assuming that newer methods must be better.

## What I Learned About Modelling

One major lesson was that volatility forecasting depends heavily on the target definition. Because next-day 30-day realised volatility is persistent, simple models already perform strongly. This helped me understand that model difficulty does not automatically lead to better forecasting.

## What Went Well

- The project became more focused over time.
- The data source was improved by switching to Hyperliquid.
- The coding pipeline became reproducible and generated saved outputs.
- The final model set became broader after the LSTM was implemented.

## What Was Difficult

- Keeping the project scope realistic
- Interpreting results fairly rather than trying to prove that machine learning must win
- Explaining the limits of realised volatility as a proxy
- Balancing code implementation time with report-writing time

## What I Would Improve

If the project continued, I would test alternative volatility windows, tune the LSTM more extensively, and possibly add richer inputs such as sentiment or intraday-based realised variance. I would also keep the daily project log and Git commits more closely aligned from the start.

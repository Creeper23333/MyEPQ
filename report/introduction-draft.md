# Introduction Draft

Cryptocurrency markets are often described as volatile because their prices can change sharply over short periods of time. Bitcoin is the most established cryptocurrency and has a longer public price history than most alternative coins, so it is a suitable case study for investigating volatility forecasting. For investors, traders, and risk managers, volatility matters because it represents uncertainty about future returns. A model that can forecast volatility more accurately may help users decide how much risk they are taking, how large a position should be, or whether market conditions have become unusually unstable.

The central question of this project is:

> How do Random Forest and Long Short-Term Memory networks compare with rolling historical volatility and GARCH(1,1) when forecasting Bitcoin volatility, in terms of accuracy, interpretability, computational practicality, and robustness?

The question is deliberately comparative. Traditional time-series models such as rolling historical volatility and GARCH(1,1) are widely used because they are relatively simple, interpretable, and designed to capture volatility clustering. Volatility clustering means that large price movements tend to be followed by further large movements, while calm periods tend to persist. This feature is particularly relevant for Bitcoin, where market stress, speculative activity, and news shocks can create sudden changes in risk.

Machine learning models offer a different approach. A Random Forest can model non-linear relationships by averaging many decision trees, while an LSTM network is designed to learn temporal patterns from sequential data. These properties make machine learning attractive for cryptocurrency forecasting, because crypto markets may contain non-linear behaviour that a basic GARCH model cannot capture. Recent studies have reported cases where neural networks or machine learning models outperform traditional econometric models, especially when additional inputs such as sentiment data are included. However, other studies find that simpler models can perform as well as complex ones depending on the forecast horizon, error metric, and specific cryptocurrency. This means that the project should not assume that machine learning will automatically be superior.

The project will therefore compare models on more than one dimension. The first dimension is accuracy, measured using MAE, MSE, and RMSE between predicted volatility and realised volatility. These metrics are important because they provide a standard way to compare forecasting errors. The second dimension is interpretability. A GARCH(1,1) model has parameters that can be explained in relation to past shocks and past volatility, while an LSTM is more difficult to interpret because its internal representation is distributed across many weights. The third dimension is computational practicality. A model that takes much longer to tune and explain may not be suitable for a small-scale EPQ unless its accuracy advantage is substantial. The final dimension is practical usefulness for risk management. A forecast is only valuable if it improves decision-making, not merely because it is mathematically complex.

The data source for the project is Hyperliquid daily Bitcoin perpetual futures candles, using the public information API. In the refreshed dataset requested on 2026-07-13, the latest available daily candle returned by the API is dated 2026-07-12. Daily closing prices are converted into logarithmic returns, and realised volatility is estimated using a rolling window. The modelling process preserves chronological order through a fixed 80/20 holdout: models are fitted on the earlier observations and evaluated on the later observations without random shuffling. This is not a walk-forward refitting design. The LSTM separately uses the final part of its training window for chronological validation and early stopping.

The expected outcome was not that machine learning must win. The corrected evidence now gives a clear multi-dimensional result: GARCH ranks first for both 14-day and 30-day targets, while LSTM improves on Random Forest but remains behind GARCH, lagged linear regression, and rolling historical volatility. This makes the project more than a technical comparison of error numbers. It evaluates whether more complex models are genuinely better tools for understanding and managing cryptocurrency volatility.

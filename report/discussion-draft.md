# Discussion Draft

The corrected results give a clearer answer than a simple "machine learning versus statistics" contest. GARCH(1,1) ranks first for both the 14-day and 30-day realised-volatility targets. Lagged linear regression and rolling historical volatility remain competitive, while LSTM ranks fourth and Random Forest fifth. The project therefore finds no forecasting-accuracy advantage for the two machine-learning models in the tested setting.

The GARCH result is theoretically plausible. Bitcoin returns exhibit volatility clustering, and GARCH directly models conditional variance as a function of recent shocks and previous variance. The project then combines the one-step conditional-variance forecast with the recent returns required to construct the next rolling realised-volatility estimate. Although this conversion is an additional modelling assumption, the fact that GARCH ranks first at both 14 and 30 days suggests that its performance is not caused only by one target-window choice.

The implementation correction is itself important evidence of critical thinking. Earlier GARCH predictions were extracted by a dataframe index that had been reset during feature engineering. Because early rows are removed when lagged features are constructed, the index was not a reliable substitute for date. The corrected pipeline aligns forecasts by date and checks that all test dates receive finite values. Reporting this change openly is preferable to preserving an attractive but invalid narrative about GARCH underperformance.

The rolling benchmark still performs well because next-day rolling volatility overlaps heavily with today's rolling volatility. Lagged linear regression improves RMSE slightly but has a higher MAE, so it is not unambiguously better on every accuracy criterion. These results show why several metrics and a transparent benchmark are needed. A model should not be called superior because of one number without considering the size and consistency of the improvement.

The machine-learning results remain informative even though they do not win. Random Forest feature importance shows that the model mainly uses current realised volatility, 30-day rolling return standard deviation, and recent volatility lags. It is therefore learning much of the same persistence signal that simpler models expose directly. LSTM performs better than Random Forest and can learn ordered sequences, but it remains behind all three statistical or interpretable alternatives at both target windows.

Interpretability changes the practical judgment. GARCH has only three fitted parameters with established volatility meanings. Linear regression provides exported coefficients, and rolling historical volatility has a one-rule explanation. Random Forest provides only a global importance summary, while LSTM has 6,049 trainable parameters distributed across recurrent and dense layers. The LSTM architecture can be documented, but this is not the same as explaining why one particular volatility forecast was produced.

Computational evidence must also be interpreted carefully. On the current machine, linear regression and the rolling benchmark are effectively immediate, GARCH takes well under one second, and both machine-learning models take several seconds to fit. The custom Random Forest is slightly slower than the small LSTM in this run, so the report should not assume that neural networks are always slower. These are implementation-specific measurements, but they show that neither machine-learning model gains accuracy in return for its additional structure and setup requirements.

The project remains limited by daily data, one exchange, one asset, a lightweight Random Forest, and a deliberately small LSTM. Realised volatility from daily returns is a proxy rather than high-frequency realised variance. The robustness check covers two target windows but not multiple market regimes, assets, or rolling refits. The results should therefore be stated as evidence for this Hyperliquid BTC sample and implementation, not as a universal rule that GARCH always beats machine learning.

For practical risk interpretation, GARCH is the most defensible current choice because it combines the lowest errors, stable ranking across both windows, a compact parameterisation, and an established explanation of volatility persistence. Rolling historical volatility remains the easiest fallback to communicate. The machine-learning models would need a consistent out-of-sample gain from richer data or stronger tuning before their reduced transparency could be justified.

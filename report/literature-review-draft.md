# Literature Review Draft

The literature for this EPQ suggests that Bitcoin volatility forecasting is a suitable setting for comparing traditional statistical models with machine learning, but it does not support a simplistic assumption that machine learning always wins. Earlier econometric work such as Bollerslev (1986) and Hansen and Lunde (2005) shows why GARCH(1,1) remains an important benchmark. GARCH models are designed to capture volatility clustering, and the literature treats them as serious forecasting tools rather than outdated baselines.

Cryptocurrency-specific studies also show why model comparison must be handled carefully. Katsiampa (2017) demonstrates that Bitcoin volatility can be analysed using GARCH-family models, while Catania, Grassi and Ravazzolo (2019) emphasise that cryptocurrency forecasting is affected by model and parameter instability. This means that a model performing well in one period may not remain best in another period, which supports using chronological validation rather than a random split.

More recent literature broadens the comparison to machine learning. Dudek et al. (2024) compare statistical models, Random Forest, LSTM, and other methods across several cryptocurrencies. Their evidence is especially useful because it resists the claim that one method is universally best. Instead, model performance depends on the asset, the forecast horizon, and the error metric. This fits the core EPQ argument that complexity should be judged in context rather than assumed to be superior.

Other studies provide evidence more favourable to neural networks. Huang, Sangiorgi and Urquhart (2024) report that machine learning techniques can outperform traditional volatility models for Bitcoin in their setting, while Shen, Wan and Leatham (2021) support the relevance of recurrent neural-network approaches for Bitcoin volatility forecasting. These sources justify keeping LSTM in the research question. However, they also create an important caution: academic neural-network implementations often use richer features, more tuning, or different volatility targets than a small-scale EPQ can reproduce.

The literature also helps frame the interpretability issue. Breiman (2001) provides the theoretical basis for Random Forest, while Hochreiter and Schmidhuber (1997) explain why LSTM is suited to sequential data. Yet interpretability sources such as Lundberg and Lee (2017) and Molnar (2025) remind us that higher predictive complexity can make a model harder to explain and trust. For an EPQ, this matters because a small numerical improvement is not automatically more useful if the model becomes much harder to justify.

The corrected results from this project fit the balanced side of the literature more closely than the strongly pro-machine-learning side. GARCH(1,1) ranks first for both 14-day and 30-day targets, while lagged linear regression and rolling historical volatility also outperform the implemented LSTM and lightweight Random Forest. This does not disprove the wider machine-learning literature. It supports the conclusion that model superiority depends on data, target definition, implementation quality, and evaluation design rather than on model family alone.
